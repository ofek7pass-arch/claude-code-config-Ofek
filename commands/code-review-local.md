# סקירת קוד מקומית (Local Code Review)

## תהליך הסקירה

### שלב 1: קביעת סקופ
שאל את המשתמש מה לסקור — קובץ ספציפי, תיקייה, או diff מסוים.

### שלב 2: איסוף מידע
זהה שפת תכנות וכללי פרויקט רלוונטיים.

### שלב 3: 4 סוכנים מקבילים
הרץ את 4 הבדיקות הבאות במקביל:
- **Bug Hunter** — שגיאות לוגיות ובאגים
- **Security Auditor** — חולשות אבטחה
- **Code Quality** — תחזוקתיות וקריאות
- **Compliance** — עמידה בכללי הפרויקט

### שלב 4: סינון ודירוג
הצג רק ממצאים עם ציון ביטחון 80+.

### שלב 5: דוח מובנה
```
🔴 Critical — חובה לתקן
🟠 Warning — מומלץ לתקן
🟡 Suggestion — שיפור אופציונלי
```

## כללים
- read-only — לא משנה קוד
- עברית עם טרמינולוגיה טכנית באנגלית
- ציונים ביטחון מדויקים עם מספרי שורה
